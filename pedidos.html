PEDIDO DEL DÍA

Misma idea que el resumen por proveedor de la app, pero acá el equipo edita el pedido del día. Depósito

Configuración o servidor

  1. En Vercel → tu proyecto → Storage → creá o conectá Postgres (suele ser Neon, con capa gratuita).
  2. En el SQL Editor de esa base, ejecutá vercel-postgres-schema.sql. Si la tabla ya existía, también vercel-postgres-migration-fecha-pedido.sql.
  3. Hacé un nuevo deploy (push a GitHub o “Redeploy”). La API vive en /api/pedidos; no hace falta ningún archivo de claves en el front.

Fecha del pedido

Los ítems quedan guardados para esa fecha. La lista de abajo se agrupa por proveedor como en «Resumen por proveedor».

Se guarda en este dispositivo. Usá «Cambiar persona» si entra otra persona al mismo celular o PC.

La lista se actualiza sola cada unos segundos para que el equipo vea los cambios sin recargar.
